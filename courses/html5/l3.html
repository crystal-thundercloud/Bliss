Урок 3. Атрибуты тегов. Всё о теге input.

В прошлом уроке мы с вами поговорили о теге img и научились вставлять картинки на страницу. Возможно, вы спросите - чего такого может быть в теге, который просто создаёт текстовое поле на странице? А мы вам ответим - тег input не так прост, как кажется.

По умолчанию тег input создаёт обычное окошко для ввода любого текста, но с помощью атрибутов его внешний вид можно изменить до неузнаваемости!

Помните задание во втором уроке? Самый первый атрибут тега input, он же - самый важный - type. Этот атрибут также является глобальным, но вот значения зависят от тега, в который его прописывают. Наш тег input обладает довольно широким спектром значений для атрибута type.

Перечислим их сразу, а затем изучим каждый подробнее.

Типы input, создающие поля для ввода текстовой информации:
text, password, email, search, url
Типы, создающие поля для ввода чисел, времени и даты:
number, date, time
Типы, создающие кнопки:
button, submit, reset, image, file
Типы выбора ответа:
radio, checkbox
Другие типы:
range, color

Как видите, их совсем немало! А если учесть ещё и типы, которые есть, но не поддерживаются большинством браузеров, то их будет ещё больше. Но мы обсудим только те, что точно будут работать почти во всех браузерах.

Input - поля ввода текстовой информации

Слово "input" переводится как "ввод", а куда обычно вводятся данные на страницах? Правильно, в текстовые поля! Но и они бывают разными.

Самое стандартное поле имеет тип text и предназначено для ввода абсолютно любой последовательности символов абсолютно любой длины. Зачастую такие поля (с ограничениями на длину) используют для ввода капчи на сайтах, а также логинов и, например, статусов в соцсетях. Вот, как выглядит такое поле:

Знакомо, не правда ли? Именно такой тип почти у всех полей проверки ваших ответов!

А вот другой тип - email. Все мы знаем, что такое почта и как обычно выглядит адрес почтового ящика. Казалось бы, внешне такое поле ничем не отличается от обычного текстового, однако оно умеет распознавать адреса почты и не позволяет отправить данные, пока адрес не будет введён правильно. Как поле находит адреса? По значку @! Попробуйте ввести обычный текст в поле ниже: оно не станет зелёным, пока вы не введёте правильный формат.

Ещё один тип, используемый для полей форм регистрации и авторизации - password. Все мы знаем, что пароль прячется за символами точек или звёздочек, а теперь вы узнали, какой тип у поля, скрывающего вводимые вами данные! Кстати, браузер автоматически распознаёт поля для ввода паролей и может предлагать запоминать введённые в них данные.

Следующий тип - search, обозначающий поле для поиска информации. Внешне оно не отличается от простого текстового поля, но при совместной работе с JavaScript начинает творить чудеса поиска по сайту.

Последнее из нашего списка поле имеет тип url, созданный специально для ввода адресов веб-сайтов. Он определяет ссылку по кодовым словам "http://" или "https://".

Input - числа, дата и время

Следующая группа полей - поля для ввода чисел. Конечно, их можно вводить и в обычное текстовое поле, но если нам ничего, кроме конкретной числовой информации, не нужно, то достаточно просто поставить нужный тип на поле input.

Тип number крайне удобен не только тем, что запрещает ввод чего-либо, кроме чисел, но и тем, что многие браузеры автоматически добавляют такому полю кнопки "вверх" и "вниз", что позволяет увеличить или уменьшить число в поле, не печатая его вручную. А если подредактировать некоторые особенные атрибуты этого поля, то можно, например, изменить шаг, на который стрелки увеличат или уменьшат число. Но и у этого типа имеется недостаток: до сих пор стрелки не реализованы на многих мобильных браузерах, поэтому, если их не добавить с помощью JavaScript, пользователь просто не поймёт, какие значения может вводить. В первом поле нет никаких ограничений на ввод, а во второе можно вводить только числа от 100001, причём шаг также равен 100001.


Теперь - один из видов полей, которые вы явно встречали, если записывались ко врачу или покупали билеты в кино: date - дата. Клик по этому полю открывает всплывающее окно выбора дня, месяца и года, но вы можете ввести дату и самостоятельно с клавиатуры.

Ну и последний тип данной категории - time. Он, как можно понять из названия, позволяет нам указать время в виде ЧЧ:ММ, причём в 24-часовом формате.

Input - кнопки

Поначалу особо и не угадать, что для поля ввода и для создания кнопки используется один и тот же тег (хотя кнопку можно сделать и с ещё одним тегом!), но это - далеко не всё, на что способен тег input.

Итак, самая обычная базовая кнопка: button. Она не имеет какой-либо встроенной функции и просто выглядит как пустая кнопка с какой-либо надписью, добавленной с помощью атрибутов.

Уже не такая обычная и до боли знакомая: file. Этот тип позволяет прикрепить файл на страницу, а если она ещё и связана с формой, то файл можно будет отправить на сервер. Ну а без привязки к серверу эта кнопка просто позволяет открывать проводник и класть на эту страницу разные файлы, которые пропадут после перезагрузки страницы. Кстати, надпись на кнопке выбора файла по умолчанию устанавливается самим браузером.

Следующие кнопки используются исключительно в формах и предоставляют две основные функции: отправку и очистку данных формы, причём для отправки существует целых две кнопки.

Одна из них - submit. В отличие от обычной кнопки, эта, находясь внутри тега form вместе с полями для ввода, при нажатии запускает автоматическую валидацию формы - например, сразу же всплывут сообщения "пожалуйста, заполните это поле".

Вторая же кнопка имеет тип image. Казалось бы, а причём тут картинки? На самом деле это нужно для создания стилей на странице: кнопка типа image имеет точно те же функции, что и кнопка типа submit, но только является картинкой, прикреплённой с помощью атрибутов. Более того, если эта картинка с прозрачным фоном, кликнуть на этот фон вы не сможете - функциональная часть кнопки будет только на тех местах, где находятся хотя бы немного непрозрачные части. Однако, такие кнопки имеют один существенный минус: нажатие на них никак визуально не отображается, если не прописать это через JavaScript. Поэтому, чтобы вы точно знали, что на следующую кнопку вы нажали, в браузере всплывёт окошко.

Ну и последняя из имеющихся кнопок - кнопка сброса формы reset. Её функция проста - полностью очищать все поля ввода.


Как мы уже сказали, кнопки можно делать не только с помощью одиночного тега input. Существует также парный тег button, создающий обычную кнопку. Внутри этого тега пишется текст, который вы бы хотели видеть на кнопке.

Input - выбор ответа

Удивительно, но на полях и кнопках наш тег input не останавливается и принимает ещё один вид - вид всем известной "галочки" или "точечки".

Тип radio обозначает "точечку", а если быть более конкретными, то он указывает на то, что из представленного списка можно выбрать только что-то одно.

Хотите узнать, как сделать так, чтобы из кучи радио выбрать можно было только один? Добавьте им атрибут name с одинаковым значением, и тогда все ваши радио будут считаться одним набором. Например, у наших радио этот атрибут прописан вот так:

name="exrd"

Почему exrd? Это сокращение от example radio. Атрибут name - глобальный, и нужен для большого количества возможностей. Для радио он обозначает их общность, а для полей ввода - имя, к которому обращается сервер на языке PHP! Поэтому, если вы не хотите потеряться в своём коде, давайте важным тегам понятные имена.

Тип checkbox - "галочка", означающая множественный выбор. Видите чекбокс - смело можете выбирать всё!

В отличие от radio, чекбоксы не нуждаются в именах для правильной работы. Но если они нужны вам для отправки информации на сервер - позаботьтесь о том, чтобы по имени было понятно, за что ваши чекбоксы отвечают.

Input - другое

В эту категорию мы добавили те типы, которые не смогли отнести к другим. Здесь их у нас всего двое, и те появились не так давно, поэтому реализация очень страдает на некоторых браузерах.

Первый в этой категории - тип range. Вы видели его на сайтах интернет-магазинов, когда меняли минимальную и максимальную стоимость в фильтрах товаров. Это бегунок, у которого слева - минимальное значение, а справа - максимальное. К сожалению, сделать его понятным без JavaScript пока не представляется возможным: браузер Chrome может отображать значения на оси бегунка, а вот Firefox - нет. К тому же, понять, что вы выбрали, если значений много, тоже очень сложно. В зависимости от браузера, у бегунка ниже вы либо увидете насечки каждые 10 значений, либо нет.

Второй тип input, которому не нашлось места среди остальных - color. Этот тип позволяет открыть стандартное окно выбор цвета, что часто используется для стилизации интерфейса на некоторых сайтах.

Возможно, вы, попробовав разные текстовые поля в этом документе, заметили, что некоторые поля для ввода специальных данных (как, например, url) принимают значения, несколько отличающиеся от предполагаемых. Это происходит потому, что встроенная валидация HTML проверяет только самые базовые элементы вводимой информации. То же поле с типом url без дополнительных параметров легко в качестве ссылки возьмёт любой текст, написанный латинскими буквами, в котором присутствует хотя бы одно двоеточие. Неизвестно, почему валидация самого HTML имеет такие низкие запросы, но вполне известно их решение. Им может быть либо всё тот же код на JavaScript, либо ещё один атрибут - pattern. Атрибут pattern создан для полей ввода текста или чисел, и представляет собой регулярное выражение - эдакую формулу, написанную с использованием символов, воспринимаемых HTML. Да-да, даже у предложений есть свои формулы! Но более подробно мы разберём их в следующем уроке.
